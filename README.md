# Onet: Omega Network Decentralized Ingress & Tunnel Manager

Onet is a unified ingress controller that seamlessly routes public internet traffic to your decentralized backend agents. It is designed to run on any Linux server and completely automates the lifecycle of Cloudflare zero-trust tunnels and FRP (Fast Reverse Proxy) tunneling.

## Architecture

Onet integrates two major components natively:

1. **Embedded FRP Server (`frps`)**: A high-performance reverse proxy that accepts incoming connections from your remote agents (`frpc`). It is compiled directly into the Onet binary.
2. **Cloudflared Auto-Manager**: Onet automatically detects your OS/Architecture, securely downloads the official `cloudflared` daemon, and manages its lifecycle as a background child process.

**Traffic Flow:**
`Internet` ➡️ `Cloudflare Edge` ➡️ `cloudflared (Child Process)` ➡️ `Onet (Embedded frps :8080)` ➡️ `Agent (frpc)` ➡️ `Your Local App`

When you migrate to a dedicated fiber line with a static IP in the future, you can simply disable Cloudflare in the configuration and open port `8080` (HTTP) and `7000` (FRP Control) on your router. No changes to the agents are required!

## Installation

1. Build the application from source:
   ```bash
   make build
   ```
2. The binary will be available at `bin/onet`.
3. To install it as a persistent `systemd` background service:
   ```bash
   sudo ./bin/onet install
   ```
4. Start the service:
   ```bash
   sudo ./bin/onet start
   ```

*(To run it interactively in the foreground for debugging, use `sudo ./bin/onet orun`)*

## Configuration

When you run Onet for the first time, it automatically generates a configuration file at `/etc/onet.yaml`.

```yaml
api_key: default-key-change-me
environment: production
port: 8080
frp_bind_port: 7000          # The port your remote agents connect to
frp_vhost_http_port: 8080    # The port cloudflared sends HTTP traffic to
frp_auth_token: change-this-token  # Security token for your agents
cf_api_token: ""             # Cloudflare API Token (Requires Zone.DNS and Account.Cloudflare Tunnels permissions)
cf_account_id: ""            # Your Cloudflare Account ID
cf_zone_id: ""               # Your Cloudflare Zone ID
cf_domain: oggree.com        # Your base domain (e.g. oggree.com)
cf_tunnel_id: ""             # Auto-generated by Onet
cf_tunnel_secret: ""         # Auto-generated by Onet
```

### Enabling Cloudflare Automation
To enable automatic Cloudflare tunneling and Zero Trust integration:
1. Provide your `cf_api_token`, `cf_account_id`, `cf_zone_id`, and `cf_domain` in `/etc/onet.yaml` (or configure them via the browser-based Arc Setup Wizard on `http://localhost:14080`).
2. **Generating the Cloudflare API Token**:
   - Navigate to your Cloudflare Dashboard -> **My Profile** -> **API Tokens**.
   - Click **Create Token** -> Select **Create Custom Token** (Start with a blank template).
   - Give the token a name and grant it the following permissions (ensuring they target your specific Account and Zone where applicable):
     - **Account Permissions**:
       - `Cloudflare Tunnel`: **Read, Write** (Required for tunnel creation & configuration)
       - `Account DNS Settings`: **Read, Write**
       - `DNS Firewall`: **Read, Write**
       - `DNS View`: **Read, Write**
       - `Access: Apps and Policies`: **Read, Write, Revoke**
       - `Access: Apps`: **Read, Write, Revoke**
       - `Access: Identity Providers`: **Read, Write**
       - `Access: SSH Auditing`: **Read, Write**
     - **Zone Permissions**:
       - `Zone.DNS`: **Read, Write** (Required to update CNAME records for domain routing)
    - Ensure the token is generated as a **User API Token** (Custom Token) under **My Profile** -> **API Tokens** (which allows both Account and Zone level permissions), NOT as an "Account API Token" under "Manage Account", and NOT as a legacy Global API Key.
3. Restart the Onet service.
4. Onet will instantly:
   - Download the `cloudflared` binary into the `bin/` folder.
   - Provision a zero-trust tunnel via the Cloudflare API.
   - Generate a secure `cf_tunnel_secret`.
   - Setup a `*.yourdomain.com` CNAME record pointing to your tunnel.
   - Start the background `cloudflared` tunnel securely.

## Hot Reloading

Onet monitors `/etc/onet.yaml` for changes. Most standard application keys will be dynamically hot-reloaded into memory without restarting the binary. *(Note: Changes to FRP or Cloudflare credentials currently require a service restart `sudo systemctl restart onet` to take effect).*

## Uninstalling

To cleanly remove the background service:
```bash
sudo ./bin/onet stop
sudo ./bin/onet uninstall
```

## Troubleshooting

### "The page you requested was not found" (FRP 404)
If you get a 404 error when trying to access subdomains or the setup wizard (`arc.yourdomain.com`):

1. **Check Connection Status**: Ensure `onet` is running and the internal FRP client is connected cleanly to the server control port (`frp_bind_port`, default `7000`). You can verify this by checking TCP loopback connections:
   ```bash
   ss -tan | grep 7000
   ```
   You should see an `ESTABLISHED` loopback connection representing the client-server link.
2. **DNS & Wildcard Conflict**: Ensure there are no conflicting A/AAAA/CNAME records in your Cloudflare DNS dashboard for `arc.yourdomain.com` or `*.yourdomain.com` that aren't pointed to your Cloudflare Tunnel. Specific DNS records always take precedence over wildcards.
3. **Authentication Scopes**: The Onet authentication webhook only intercepts `Login` operations. Remote agents do not require `additionalScopes` settings in their `frpc.toml` configuration to establish work tunnels.
